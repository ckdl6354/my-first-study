# 🔤 Developer Naming Cheatsheet

## 1. 클래스(Class) 역할별 접미사
* **`~Controller`**: 요청 및 응답 처리 (`MemberController`)
* **`~Service`**: 비즈니스 로직 처리 (`OrderService`)
* **`~Repository` / `~Dao`**: DB 접근 데이터 처리 (`UserRepository`)
* **`~Dto`**: 데이터 전송 객체 (`UserJoinDto`)
* **`~Impl`**: 인터페이스 구현체 (`PaymentServiceImpl`)

## 2. 메서드(Method) 동사 활용 규칙
* **`get~` vs `find~`**: 무조건 조회 (`getUserName`) vs 검색 목적 (`findUserById`)
* **`create~` vs `register~`**: 객체 생성 (`createOrder`) vs 시스템 등록 (`registerMember`)
* **`modify~` vs `update~`**: 일부 수정 (`modifyProfile`) vs 상태 갱신 (`updateStatus`)
* **`check~` vs `is~`**: 검사 행위 (`checkPassword`) vs boolean 반환 (`isValid`, `isAdmin`)

## 3. 변수(Variable) 단골 단어 매칭
* **식별자**: 고유번호 `id` | 이름 `name` | 제목 `title`
* **수량**: 총개수 `count` / `total` | 주문수량 `quantity` (`qty`)
* **시간**: 생성일 `createdAt` | 수정일 `updatedAt`
* **상태**: 성공여부 `isSuccess` | 삭제여부 `isDeleted`
* **임시**: 임시 데이터 `temp` (`tmp`)

## 4. 네이밍 금지 규칙
* **한 글자 금지**: `int a;`, `String s;` (반복문 인덱스 `i` 제외)
* **한글 발음 금지**: `String juhwawon;`(X) => `String member;`(O)
